## Project Overview:

The **Life Hack or Urban Myth App** is a mobile application developed as part of an assignment in the Introduction to Mobile Applications Development (IMAD5112) subject. This application was created using **Kotlin** and **Android Studio**. The app's primary purpose is to test the user's knowledge by presenting statements and asking them to decide whether each one is a real life hack or just an urban myth.

The app was developed to meet the requirements of the assignment, which includes creating a functional mobile app and utilizing GitHub for version control and CI/CD automation using GitHub Actions.

---

## Purpose and Features

### Purpose:
So basically the app is meant to help users test their common sense and learn useful real world shortcuts. A lot of people struggle to tell the difference between genuine productivity tips and urban myths that have gone viral. So the app gives you a fun flashcard style quiz to help you figure out what is real and what is not.

### Key Features:
- Feature 1: A Welcome Screen with a brief description of the app and a Start button
- Feature 2: A Flashcard Question Screen that shows a statement and lets you answer Hack or Myth
- Feature 3: Instant feedback after each answer telling you if you were correct or wrong
- Feature 4: A Score Screen at the end showing your total score and personalised feedback
- Feature 5: A Review Screen that shows all the statements and their correct answers
- Feature 6: A Play Again button to restart the quiz from the beginning

These features work together to create a fun and educational experience that helps users learn the difference between real life hacks and urban myths.

## Design Considerations

The design of the **Life Hack or Urban Myth App** was based on the following key considerations:

1. **User Experience (UX):** The app was designed with the intention of providing a straightforward and easy to navigate user interface. The goal was to ensure that any user could open the app and immediately understand how to use it without any prior guidance.

2. **Responsiveness:** The app was developed to function correctly across a range of Android screen sizes, ensuring that the layout and all UI elements remain consistent and properly displayed on different devices.

3. **Simplicity:** The design follows a clean and minimal approach, using an orange background and light red cards to create a visually appealing interface without overwhelming the user with unnecessary elements.

4. **Performance:** The application logic was kept lightweight and efficient through the use of Boolean values and if/else conditional statements, ensuring that the app responds quickly to user input with no noticeable delays.

## GitHub and GitHub Actions

This project was managed using **GitHub** for version control, where all code changes were committed and pushed regularly. GitHub enabled me to keep track of all changes made to the project throughout the development process and maintain the integrity of the codebase.

### GitHub Actions:
I utilized **GitHub Actions** to automate the build and deployment process. This includes:

- Running automated **tests** to ensure the app's functionality is working as expected.
- Compiling the app into **APK** and **AAB** files, which are the formats required for distribution.
- Uploading these build artifacts to GitHub for easy access and submission.

The workflow ensures that the project is automatically built and tested every time changes are pushed, simplifying the process of delivering the final APK/AAB files for submission.
